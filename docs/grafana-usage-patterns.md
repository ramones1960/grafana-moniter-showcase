# Grafana の利用パターン整理

「Grafana の様々な利用方法」を体系的に整理する。本リポジトリの 3 本柱に対応。

## ① 各種 DB との連携基盤としての Grafana

Grafana は 150+ のデータソースプラグインを持つ「**可視化のためのハブ**」。
DB 側に手を入れず、読み取り専用で横断的にダッシュボードを作れるのが価値。

```
        ┌─────────────────────────────┐
        │           Grafana           │
        │   (datasource provisioning) │
        └──┬─────────┬─────────┬──────┘
           │ PromQL  │ Flux    │ SQL
       ┌───▼───┐ ┌───▼────┐ ┌──▼─────────┐
       │ Prom  │ │InfluxDB│ │ TimescaleDB │
       └───────┘ └────────┘ └─────────────┘
```

- 用途別に DB を選ぶ（メトリクス=Prometheus / 高頻度=InfluxDB / 台帳=RDB）
- 1 つのダッシュボードに**複数データソースのパネルを混在**できる（Mixed datasource）
- `compare-db-oss` の各 DB をデータソースとして追加すれば、DB ごとの Grafana 適合性も比較評価できる

## ② Web アプリのフロントエンドを集約する用途

複数の社内システム・監視画面を「1 つの窓」に集約するポータルとしての Grafana。

| 手段 | 仕組み | メリット | 注意点 |
|---|---|---|---|
| iframe 埋め込み | パネル/ダッシュボードの share URL を iframe | 実装ゼロ | 認証・CORS・`allow_embedding` 設定 |
| Public dashboards | 認証不要の公開 URL | 外部公開ステータスに最適 | 機密データ厳禁 |
| Grafana Scenes | React 用 SDK でパネルを部品として利用 | 自社 UI にネイティブ統合 | 実装コスト |
| Snapshot | 静的スナップショット共有 | 障害報告に便利 | リアルタイム性なし |

→ 宇宙開発では「ミッション運用ポータル」「対外公開のミッションステータスページ」に直結。

## ③ 各種リソース監視

最も一般的な使い方。exporter がメトリクスを公開 → Prometheus がスクレイプ → Grafana が可視化。

| 監視対象 | exporter | 主なメトリクス |
|---|---|---|
| ホスト | node_exporter | CPU/メモリ/ディスク/NW |
| コンテナ | cAdvisor | コンテナ別リソース |
| 外形監視 | blackbox_exporter | エンドポイント死活・遅延 |
| アプリ | 各言語クライアント | カスタム業務メトリクス |

本デモではシミュレータが `prometheus_client` で**衛星ヘルスを業務メトリクス**として公開し、
「アプリのリソース監視」と「衛星の業務監視」を同じ Prometheus 流儀で扱う例を示す。
